# Mini App для ввода активностей + буферная запись в Sheets

Дата: 2026-07-06. Статус: утверждено в брейншторме, прототип механики пройден.

## Контекст и проблема

Полгода ежедневного использования выявили два ограничения текущего бота:

1. **Задержка ввода ~1с+ на каждое нажатие.** Причины: `asyncio.sleep(0.5)` в
   `handle_stars_selection`, синхронный `sheets.write_values()` (3–5 HTTP-запросов
   к Google) блокирует event loop, аналитика делает до 30 запросов подряд.
   Ошибки записи глотаются `except: pass` — возможна молчаливая потеря данных.
2. **Потолок UX инлайн-кнопок.** Вечерний ритуал — визуальный проход по всем
   ~32 активностям («делал ли я это?»). Инлайн-клавиатура не показывает всё
   сразу и требует переходов между экранами.

Требование-константа: **Google Sheets остаётся первичной БД** (надёжность,
переносимость, независимость от сервера).

## Решение (утверждённый подход)

Telegram Mini App для ввода + буферная запись: журнал на диске развязывает UI
от Sheets. Бот остаётся для напоминаний, /stats и как fallback-ввод.

### Механика ввода (проверена прототипом `prototype/input-ux.html`)

Победил вариант **B — «тап → пресеты»**:

- Сетка плиток всех активностей (2 колонки, порядок недавнего использования)
- Тап по плитке → под ней раскрывается панель пресетов часов
- Пресеты: **0.5–3.5 шагом 0.5, дальше целыми до 15** (19 кнопок) + ✏️ custom + ✕ сброс
- Тап по пресету → панель сворачивается, плитка помечена значением
- Повторный тап по заполненной плитке → снова пресеты (перевыбор)

Степперы (варианты A и C) отвергнуты.

## Архитектура

```
Telegram ──── бот (PTB, polling) ────┐
                                     ├── WriteBuffer ── journal.jsonl ── sync-воркер ──> Google Sheets
Mini App ──── HTTP API (FastAPI) ────┘        │                                            │
  (статика с того же сервера)                 └── DayCache <────── стартовая сверка ───────┘
```

Один процесс, один контейнер. В одном asyncio loop: PTB-бот (ручной lifecycle
`initialize/start/updater.start_polling` вместо блокирующего `run_polling`),
uvicorn с FastAPI, sync-воркер.

> Рассмотрен и отклонён webhook-режим бота (как в budget-bot): polling не
> зависит от доступности caddy/HTTPS и не требует переделки бота.

### Компоненты

**1. HTTP API (FastAPI)**

| Ручка | Что делает |
|---|---|
| `GET /api/day/{date}` | Значения дня: DayCache + несинканный хвост журнала поверх |
| `POST /api/entry` | `{date, hobby, hours}` → журнал + кэш → 200 (<50мс) |
| `GET /api/hobbies` | Категории в порядке истории + display-имена из алиасов |
| `GET /` (статика) | `frontend/index.html` Mini App |

Auth — паттерн budget-bot (проверен на этой же инфре): заголовок
`Telegram-Init-Data`, HMAC-SHA256 валидация подписи (`WebAppData` + bot token),
allowlist user_id из env `ALLOWED_USER_IDS`. FastAPI dependency на каждой ручке.

**2. WriteBuffer** — единственный путь записи для бота И Mini App.
Append-only `data/journal.jsonl`, строка:

```json
{"ts": "2026-07-06T21:15:03+03:00", "date": "2026-07-06", "hobby": "игры", "hours": 2.0, "source": "miniapp"}
```

`data/journal.offset` — число слитых в Sheets строк. Append → мгновенный
возврат. Никаких прямых вызовов `SheetsManager.write_values` из хендлеров.
При старте, если offset == числу строк журнала, файл усекается (компактация,
чтобы не рос годами). Битая строка журнала (недописанная при падении)
пропускается с warning, не валит воркер.

**3. Sync-воркер** — asyncio-задача. Будится по событию новой записи (не по
таймеру), батчит накопившееся (последнее значение на пару hobby+date
побеждает), пишет в Sheets через `asyncio.to_thread`, retry с экспоненциальным
backoff (Sheets недоступен / rate limit), после успеха сдвигает offset.
Рестарт контейнера → доигрывает несинканный хвост журнала.

**4. DayCache** — `data/cache/days.json`, снапшот значений **последних 7
дней**. Обновляется при записи и при чтении из Sheets. При старте процесса —
сверка всех 7 дней с Sheets (Sheets истина: правки руками в таблице
подтянутся). Даты старше 7 дней в кэше не живут — всегда прямое чтение из
Sheets (редкий случай, задержка приемлема).

**5. Фронт Mini App** — `frontend/index.html`, vanilla JS/CSS без сборки,
переносится из победившего прототипа B:

- `telegram-web-app.js`: initData, цвета из `themeParams`
- Переключатель даты: Сегодня / Вчера / 📅 календарь; правило «до 6 утра =
  вчера» (`date_for_time`) действует и здесь: `GET /api/hobbies` возвращает
  поле `default_date`, фронт открывается на нём
- Custom-ввод (✏️): валидация 0–24 с шагом 0.5, точка и запятая (текущее
  поведение бота)
- Optimistic UI: тап → плитка помечена сразу, POST фоном; при ошибке сети —
  откат плитки + сообщение
- Футер: итого часов, число активностей, индикатор синка — ответы
  `POST /api/entry` и `GET /api/day/{date}` содержат поле `queue_pending`
  (число несинканных записей)

**6. Изменения бота**

- Кнопка открытия Mini App: menu button + кнопка в `/quick`
- Инлайн-флоу остаётся как fallback, но пишет через WriteBuffer
- Удаляется `asyncio.sleep(0.5)` из `handle_stars_selection`
- Все чтения Sheets в хендлерах → `asyncio.to_thread`
- Аналитика (`get_week_data`, топ-3): одно `get_all_values()` вместо 7–30
  запросов `get_day_data`
- Напоминания (scheduler) — без изменений, читают через тот же SheetsManager

## Поток данных

**Запись:** тап → optimistic UI → `POST /api/entry` → append журнал + кэш →
200 → воркер (по событию) → batch в Sheets → offset++. Повторная запись той же
пары hobby+date = перезапись значения (текущее поведение сохраняется).

**Чтение:** `GET /api/day/{date}` → кэш + оверлей журнала → мгновенно.

## Обработка ошибок

| Сбой | Поведение |
|---|---|
| Sheets недоступен / лимиты | UI «✓ записано» (журнал на диске), воркер ретраит, индикатор очереди |
| Контейнер умер до синка | Журнал на volume, доигрывается при старте |
| POST не дошёл (сеть телефона) | Откат плитки + ошибка в UI — единственный случай «не записалось» |
| Ручные правки в таблице | Стартовая сверка перечитывает; Sheets истина |
| Невалидный initData | 401/403, ручки закрыты allowlist'ом |

## Деплой (orion.artfaal.ru)

Инфра-паттерн — как budget-bot (`/var/docker/compose/budget-bot`):
caddy-docker-proxy, labels + external network. DNS A-запись
`hobby.artfaal.ru → 206.245.129.248` уже заведена (DNS only — Caddy сам
получит Let's Encrypt сертификат).

Дополнения в `docker-compose.yml`:

```yaml
services:
  hobby-tracker:
    # ... существующее ...
    environment:
      # ... существующее ...
      - WEBAPP_URL=https://hobby.artfaal.ru
      - ALLOWED_USER_IDS=${ALLOWED_USER_IDS}
    labels:
      caddy: hobby.artfaal.ru
      caddy.reverse_proxy: "{{upstreams 8000}}"
    networks:
      - caddy
      - hobby-network

  caddy:
    external: true
```

Плюс: uvicorn слушает `0.0.0.0:8000` внутри контейнера (порт наружу не
пробрасывается — caddy ходит по docker-сети). В BotFather Mini App URL задаётся
кодом через `WebAppInfo(url=WEBAPP_URL)` — отдельной настройки не нужно
(паттерн budget-bot, `main.py:269`).

## Тестирование

- **pytest на ядро надёжности** (единственное место с риском потери данных):
  WriteBuffer — append/offset/replay после «падения», компактация батча
  (последнее значение побеждает), оверлей журнала на кэш, парсинг журнала
  с битой строкой
- **API + auth**: интеграционные тесты с фейковым `initData` (валидная/невалидная
  подпись, чужой user_id)
- **UI + Sheets-синк**: ручной прогон на стенде, `ui-tester` на стадии Validate
- **Бот-fallback**: ручная проверка старого флоу

## Scope v1 / не в scope

**В v1:** Mini App (ввод + сегодня/вчера/календарь), WriteBuffer + воркер +
DayCache, починка задержек бота, batch-аналитика, деплой на hobby.artfaal.ru.

**Не в scope v1:** статистика/графики в Mini App, добавление новых активностей
из Mini App (через бот), управление алиасами/напоминаниями в Mini App,
мульти-юзер.

## Критерий приёмки

Вечерний ритуал: открыть Mini App из бота, пройти по ~30 плиткам, проставить
часы за день тапами без ощутимых задержек (<100мс реакция плитки), закрыть.
Данные оказываются в Google Sheets (сразу или после восстановления связи).
Старый бот-флоу работает без `sleep` и блокировок. Убитый посреди синка
контейнер не теряет записи.

## Пострелизные изменения

Факты по итогам реализации (прод — https://hobby.artfaal.ru). Расхождения с
описанием выше, зафиксированные для истории:

- **Индикатор синка — в шапке, не в футере.** В спеке итог дня и индикатор
  очереди планировались в футере; реализованы в sticky-шапке (`<header>`,
  `frontend/index.html`): строка со счётчиком часов/активностей и статусом
  записи в таблицу.
- **Добавлена ручка `GET /api/queue`** (сверх четырёх из спеки). Лёгкий статус
  очереди `{queue_pending}`; фронт опрашивает её после записи, пока не увидит 0
  (снимок в ответе `POST /api/entry` сделан до слива воркером и всегда ≥1).
  Статусы в UI: «⏳ пишу в таблицу… (N)» → «✓ всё в таблице». Итоговый список
  ручек: `GET /api/hobbies`, `GET /api/day/{date}`, `POST /api/entry`,
  `GET /api/queue` + раздача статики.
- **Стартовая сверка кэша реализована как `runtime.reconcile_cache()`** (находка
  валидатора): при недоступности Sheets сверка НЕ затирает кэш пустотой —
  `_fetch_days_strict` бросает исключение, кэш остаётся как есть (см. предупреждение
  в логе), затем `cache.prune()` удаляет даты старше 7 дней. Вызывается из
  `main.py:amain()` до старта бота.
