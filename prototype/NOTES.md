# PROTOTYPE — механика ввода (throwaway)

**Вопрос:** какая механика ввода часов по ~30 активностям быстрее/удобнее
для вечернего ритуала в будущем Telegram Mini App.

**Варианты** (`input-ux.html`, переключение стрелками внизу или `?variant=`):

- **A — Степперы везде**: на каждой плитке [− часы +], всё видно сразу, ноль переходов.
- **B — Тап → пресеты**: тап по плитке раскрывает ряд [0.5][1][1.5][2][3][4][5][6][8][✏️][✕].
- **C — Гибрид**: пустая плитка тап→пресеты; заполненная — значение + мини-степперы ±0.5.

**Запуск:** `open prototype/input-ux.html` (телефонную ширину эмулирует, состояние
общее между вариантами — можно переключаться посреди «ритуала»).

## Вердикт (2026-07-06)

**Победил B — «тап → пресеты».** Степперы (A, C) не нужны.

Уточнение по пресетам: 0.5–3.5 шагом 0.5, дальше целыми до 15 включительно
(итого 19 кнопок + ✏️ custom + ✕ сброс). Прототип обновлён — проверить, что
